## MockWorld Tests

A testing framework for AI agents. Built on mock servers. Implements the [MockWorld Test paradigm](/mockworld-tests/the-paradigm) introduced by Peter Nsaka.

<Tabs>
  <Tab title="The Problem">
    AI agents are non-deterministic:

    ```
    Run 1: Agent takes 3 steps → creates refund
    Run 2: Agent takes 7 steps → creates refund
    Run 3: Agent takes 5 steps → creates refund
    ```

    Traditional tests fail because they assert on specific steps.

    And testing in production is dangerous — an agent might process 10,000 refunds before you catch the bug.
  </Tab>
  <Tab title="The Solution">
    Assert on outcomes, not paths:

    ```python
    world = mockworld("Refund test", services=["stripe", "shopify"])

    with world.run():
        agent.invoke("Process refund for order #1234")
        # Agent decides what to do

    # See what happened
    world.observe()
    # => "Agent created refund of $50 in Stripe"

    # Assert the outcome
    world.assert("exactly one refund was created")
    # Passes regardless of how many steps the agent took
    ```
  </Tab>
</Tabs>

## Why Mokra

<AccordionGroup>
  <Accordion title="vs. Sandbox accounts (Stripe test mode, Shopify partners, etc.)">
    **Sandboxes:** One credential per service. Rate limits. Polluted state from old tests. Different setup for each.

    **Mokra:** One credential for 800+ services. No rate limits. Fresh state every time. Same setup for all.
  </Accordion>

  <Accordion title="vs. Mocking libraries (VCR, WebMock, responses)">
    **Mocking libraries:** Record-and-playback. Stateless. Manual cassette management. Breaks when APIs change.

    **Mokra:** Live mock servers. Stateful (create a payment, then refund it). Auto-maintained. Behaves like the real API.
  </Accordion>

  <Accordion title="vs. LangSmith (for AI agent testing)">
    **LangSmith:** Traces LangChain callbacks. Shows LLM calls and tool invocations. Debug-focused.

    **Mokra:** Operates at HTTP layer. Catches ALL network calls (not just LangChain). Outcome-focused assertions.

    *Use both: LangSmith for debugging reasoning, Mokra for testing outcomes.*
  </Accordion>
</AccordionGroup>
