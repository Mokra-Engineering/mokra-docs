# API Reference

The Mokra API allows you to create MockWorlds, access mock servers, and run assertions programmatically.

## Base URL

```
https://api.mokra.ai/api
```

## Authentication

All requests require an API key in the `X-API-Key` header:

```bash
curl https://api.mokra.ai/api/worlds \
  -H "X-API-Key: mk_live_your_key"
```

## Core concepts

### Mock Servers

Mock servers are replicas of real-world APIs. Each service (Stripe, Shopify, etc.) has its own mock server that returns realistic responses.

### Worlds

A World is a container that groups mock servers together for a test. It provides:
- Isolated state
- Observation recording
- Assertion evaluation

### Observations

Observations are human-readable records of what happened during a test. They describe API calls in plain English.

### Assertions

Assertions verify that expected outcomes occurred, regardless of the path taken to get there.

## Quick example

```bash
# 1. Create a world
curl -X POST https://api.mokra.ai/api/worlds \
  -H "X-API-Key: mk_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "name": "My test",
    "services": ["stripe", "shopify"]
  }'

# Response:
# {
#   "id": "world_abc123",
#   "mock_servers": {
#     "stripe": { "id": "ms_xyz", "base_url": "..." },
#     "shopify": { "id": "ms_abc", "base_url": "..." }
#   }
# }

# 2. Make API calls through the mock server
curl https://api.mokra.ai/mock/stripe/v1/charges \
  -H "X-API-Key: mk_live_..." \
  -H "X-World-Id: world_abc123" \
  -d "amount=5000&currency=usd"

# 3. Get observations
curl https://api.mokra.ai/api/worlds/world_abc123/observations \
  -H "X-API-Key: mk_live_..."

# 4. Run assertions
curl -X POST https://api.mokra.ai/api/worlds/world_abc123/assert \
  -H "X-API-Key: mk_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "assertions": ["a charge was created"]
  }'
```
